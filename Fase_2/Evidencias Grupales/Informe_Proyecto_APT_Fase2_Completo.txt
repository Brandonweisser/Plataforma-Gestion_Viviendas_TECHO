1. DESCRIPCIÓN DEL PROYECTO APT

1.1 Nombre del Proyecto
Sistema de Gestión de Viviendas - TECHO

1.2 Descripción General
El proyecto consiste en el desarrollo de una plataforma web integral que permite a TECHO-Chile gestionar el ciclo completo de las viviendas sociales bajo la normativa DS49, desde la planificación del proyecto hasta el seguimiento posterior a la entrega.

La plataforma ha evolucionado significativamente desde su concepción inicial, implementando una arquitectura modular profesional que incluye módulos especializados para la gestión de incidencias, formularios de recepción digital, evaluaciones de postventa automatizadas y un sistema completo de reportería.

El sistema facilita la trazabilidad y el control de calidad en todos los procesos, optimizando la coordinación entre beneficiarios, técnicos y administradores en todas las etapas del proceso de vivienda.

1.3 Funcionalidades Principales Implementadas

• Control de Usuarios y Accesos
  - Autenticación segura con roles diferenciados
  - Perfiles personalizados para cada tipo de usuario
  - Dashboard adaptado según nivel de acceso

• Administración de Proyectos
  - Registro y seguimiento de proyectos habitacionales
  - Gestión de cronogramas y entregas
  - Control de inventario de viviendas por estado

• Proceso de Recepción Digital
  - Formularios digitales de verificación por categorías
  - Registro fotográfico integrado
  - Flujo de aprobación y observaciones

• Sistema de Incidencias Avanzado
  - Reporte directo por parte de beneficiarios
  - Asignación y seguimiento técnico automatizado
  - Historial completo de gestiones y resoluciones
  - Clasificación por prioridad y categoría

• Evaluación de Postventa Automatizada
  - Formularios de satisfacción periódicos
  - Generación automática de reportes PDF
  - Indicadores de calidad y seguimiento
  - Dashboard de métricas y estadísticas

• Gestión Documental Completa
  - Almacenamiento seguro en la nube
  - Generación automática de reportes
  - Trazabilidad completa de documentación
  - Métricas de desempeño y calidad

1.4 Stack Tecnológico Implementado

Frontend:
• React 18 - Biblioteca de interfaz de usuario moderna
• React Router - Navegación entre páginas fluida
• Tailwind CSS - Framework de estilos utility-first
• Axios - Cliente HTTP para comunicación con API

Backend:
• Node.js - Entorno de ejecución JavaScript
• Express.js - Framework para API REST modular
• JSON Web Tokens (JWT) - Sistema de autenticación seguro
• Multer - Procesamiento de archivos multimedia
• html-pdf-node - Generación de documentos PDF

Base de Datos:
• PostgreSQL - Base de datos relacional robusta (via Supabase)
• Supabase - Plataforma de base de datos como servicio
• Supabase Storage - Almacenamiento de archivos en la nube

Herramientas de Desarrollo:
• ESLint - Análisis de código JavaScript
• Jest - Framework de testing automatizado
• dotenv - Gestión de variables de entorno
• GitHub - Control de versiones y colaboración

1.5 Arquitectura Modular Implementada

El backend ha sido completamente refactorizado siguiendo patrones de arquitectura profesional para mejorar mantenibilidad, escalabilidad y colaboración en equipo:

backend/
├── controllers/          # Lógica de negocio separada por funcionalidad
│   ├── authController.js      # Autenticación y autorización
│   ├── adminController.js     # Gestión administrativa
│   ├── beneficiarioController.js # Funciones para beneficiarios
│   └── tecnicoController.js   # Gestión técnica de incidencias
├── middleware/           # Middleware reutilizable
│   └── auth.js               # Verificación JWT y manejo de roles
├── models/              # Acceso a datos y lógica de base de datos
│   ├── User.js               # Gestión de usuarios y autenticación
│   ├── Project.js            # Gestión de proyectos habitacionales
│   ├── Housing.js            # Gestión de viviendas y asignaciones
│   ├── Incidence.js          # Gestión de incidencias y reportes
│   └── PasswordRecovery.js   # Sistema de recuperación de contraseñas
├── routes/              # Definición modular de rutas API
│   ├── auth.js               # /api/* (registro, login, recuperación)
│   ├── admin.js              # /api/admin/* (gestión administrativa)
│   ├── beneficiario.js       # /api/beneficiario/* (funciones beneficiario)
│   └── tecnico.js            # /api/tecnico/* (gestión técnica)
├── services/            # Servicios externos
│   ├── EmailService.js       # Envío de correos electrónicos
│   └── PosventaPDFService.js # Generación de documentos PDF
└── utils/               # Utilidades y validaciones centralizadas
    └── validation.js         # Validaciones reutilizables (RUT, email, etc.)

9.2 Desafíos Superados y Lecciones Aprendidas

Desafíos Técnicos Significativos:

Refactorización de Arquitectura Monolítica
Inicialmente, nuestro backend se desarrolló como una aplicación monolítica que, aunque funcional, presentaba limitaciones para el mantenimiento y la escalabilidad. La refactorización hacia una arquitectura modular nos enseñó:
• La importancia de la planificación arquitectónica desde el inicio del proyecto
• Técnicas para migrar código legacy manteniendo funcionalidad completa
• Valor de la separación de responsabilidades para el trabajo en equipo
• Beneficios a largo plazo de invertir tiempo en calidad arquitectónica

Lección clave: La calidad del código y la arquitectura son inversiones, no gastos.

Optimización de Rendimiento para Conexiones Limitadas
Reconociendo que los beneficiarios de TECHO-Chile pueden tener acceso limitado a internet, implementamos optimizaciones específicas:
• Compresión de assets y lazy loading para reducir tiempos de carga iniciales
• Caching estratégico en frontend y backend para minimizar solicitudes redundantes
• Diseño responsive optimizado para dispositivos móviles con conexiones lentas
• Implementación de progressive web app features para funcionalidad offline básica

Lección clave: El contexto social y tecnológico de los usuarios finales debe ser una consideración central en el diseño técnico.

Seguridad Robusta para Datos Sensibles
La gestión de información personal de familias vulnerables requirió implementación de seguridad multicapa:
• Cifrado de contraseñas con algoritmos seguros y salt personalizado
• Control de acceso granular con roles y permisos específicos
• Rate limiting para prevenir ataques de fuerza bruta y DOS
• Auditoría de accesos y trazabilidad completa de operaciones sensibles

Lección clave: La seguridad no es una funcionalidad adicional, sino un requerimiento fundamental que debe integrarse desde el diseño inicial.

Desafíos Metodológicos y de Proceso:

Adaptación de XP a Proyecto con Cliente Externo
